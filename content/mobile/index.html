The mobile web is everywhere!

The web, in your pocket

Nearly every mobile device that ships today has a great modern browser built in, and users are browsing them web with them more and more. In fact, mobile web usage is set to over take traditional web use in the not too distant future.

Mobifying your site doesn't always mean that you need to throw everything away and start over. Just make sure you steer clear of the cardinal sins of mobile web development.

How does your site look on mobile?

Find out how your site behaves on mobile with PageSpeed Insights and see what you can do to improve it.

User Experience

Creating an experience for a small device requires different thinking. For example, hover effects are almost useless on touch displays, filling out complex forms without a keyboard, annoying, and small screens make tiny UI elements useless. When done right, the mobile web can be just as easy to use.

Key issues

  • Use a mobile-first responsive web design
  • Build touch friendly interfaces
  • Make it easier for users to fill in forms, enter usernames or provide other information
  • Use high dpi images for the appropriate devices

Performance

Mobile devices typically have less computing power then their desktop counterparts. Add to that the unique characteristics of mobile networks with variable speeds and latency, building sites that are fast and responsive is vital. If your site takes too long to load, your user is gone.

Key issues

  • Prioritize visible content
  • Minify resources like HTML, CSS and JavaScript
  • Enable compression
  • Optimize images
  • Enable browser caching
  • Eliminating render blocking resources
  • Optimizing mobile sites with TCP in mind
